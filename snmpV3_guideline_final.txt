1. ФУНДАМЕНТАЛЬНАЯ ТЕОРИЯ И ИНФОРМАЦИОННЫЙ ОБМЕН SNMPV3

SNMPv3 (Simple Network Management Protocol version 3) — это стандарт мониторинга, ориентированный на безопасность. Он использует модель безопасности USM (User-based Security Model) для защиты данных и VACM (View-based Access Control Model) для разграничения прав доступа.
Ключевые RFC (стандарты протокола):

* RFC 3411: Описывает общую архитектуру SNMP. Вводит понятия «сущностей» (SNMP entities) и модульную структуру протокола.
* RFC 3412: Определяет обработку сообщений и диспетчеризацию. Описывает инкапсуляцию PDU в транспортные пакеты.
* RFC 3414 (USM): Модель безопасности на основе пользователей. Алгоритмы аутентификации (SHA, MD5) и шифрования (AES, DES).
* RFC 3415 (VACM): Модель управления доступом на основе «видов» (Views). Ограничение доступа к веткам дерева OID.
* RFC 3416: Определяет операции протокола (Get, Set, Response, Trap, Inform).
* RFC 2578 (SMIv2): Определяет структуру управляющей информации и базовые типы данных.

Криптография и хеширование в SNMPv3:

* Аутентификация (Authentication):
* SHA-1 (SHA): Хеш 160 бит. Надежный минимум.
   * SHA-2 (SHA-224/256/384/512): Современный стандарт (RFC 7860).
   * MD5: Устаревший протокол, используется только для совместимости.
* Шифрование (Privacy):
* AES: Основной стандарт. Чаще всего AES-128.
   * DES / 3DES: Устаревшие протоколы, не рекомендуются.

Роль EngineID и Контекстов:

* msgAuthoritativeEngineID: Уникальный серийный номер агента (Octet String, 5–32 байта). Ключ шифрования вычисляется как f(Password, EngineID).
* Роль в Трапах: Приемник (Zabbix) обязан иметь в своем конфиге EngineID отправителя, иначе трап будет отброшен.
* contextEngineID / contextName: Позволяют разделять области данных внутри одного агента (например, разные инстансы приложений).

Этапы информационного обмена:

   1. Discovery: Менеджер шлет пустой запрос. Агент отвечает сообщением REPORT PDU, сообщая EngineID, Boots и Time.
   2. Synchronization: Менеджер синхронизирует время (окно +/- 150 сек) и вычисляет ключи.
   3. Payload: Обмен зашифрованными данными.

Структура полей пакета (PDU):

* msgVersion: Integer (1б). Для v3 всегда значение 3.
* msgID: Integer (4б). Уникальный ID запроса.
* msgMaxSize: Integer (4б). Лимит размера пакета.
* msgFlags: Octet (1б). 0x03 (Auth и Priv), 0x01 (только Auth).
* msgSecurityModel: Integer (1б). Для USM значение 3.
* msgAuthEngineID: Octet (5-32б). HEX-код агента.
* msgEngineBoots: Integer (4б). Счетчик холодных стартов (защита от replay).
* msgEngineTime: Integer (4б). Секунды с последнего старта.
* msgUserName: String (1-32б). Имя пользователя USM.
* msgAuthParams: Octet (12б). HMAC-SHA хеш (подпись).
* msgPrivParams: Octet (8б). Вектор инициализации (IV/Salt) для AES.
* contextEngineID: Octet (5-32б). ID владельца контекста.
* contextName: String (0-32б). Имя инстанса данных.


2. ТИПЫ ДАННЫХ SNMP (ASN.1) И КОНВЕРСИЯ


* i (Integer): Знаковое 32 бит. Границы: -2 147 483 648 до 2 147 483 647.
* u (Unsigned): 32 бит без знака. Границы: 0 до 4 294 967 295.
* c (Counter32/64): Накопительный счетчик.
* g (Gauge32): Текущее значение (0 до 4.2 млрд). При превышении замирает.
* s (String): Octet String. Границы до 65 535 байт.
* t (TimeTicks): Время в 0.01 сек. До 497 дней.
* a (IpAddress): Строго 4 байта (IPv4).

Методы конверсии в CLI:

* Date (Hex в ISO): printf "%d-%02d-%02d %02d:%02d\n" 0x${HEX:0:4} 0x${HEX:4:2} 0x${HEX:6:2} 0x${HEX:8:2} 0x${HEX:10:2}
* IP (Hex в Dec): echo $HEX | awk '{ printf "%d.%d.%d.%d\n", strtonum("0x"$1), strtonum("0x"$2), strtonum("0x"$3), strtonum("0x"$4) }'


3. ПОДГОТОВКА ХОСТА ПОД МОНИТОРИНГ ZABBIX
Установка: apt install snmpd snmp-mibs-downloader -y && download-mibs
Активация MIB: В /etc/snmp/snmp.conf закомментируйте строку #mibs :
Базовый конфиг /etc/snmp/snmpd.conf:

   agentAddress udp:161
   sysLocation "Data Center 1"
   view systemview included .1
   rouser myuserv3 authPriv -V systemview
   5. Создание пользователя:
   systemctl stop snmpd
   net-snmp-config --create-snmpv3-user -a SHA -A AuthPass123 -x AES -X PrivPass123 myuserv3
   systemctl start snmpd
   
4. КОНФИГУРАЦИЯ snmpd.conf (VACM, РОЛИ И ТРАПЫ)

agentAddress udp:161
engineID 0x8000000001020304
## Ролевая модель VACM
view all_view included .1
group MyGroup usm myuserv3
access MyGroup "" usm authPriv exact all_view all_view none
## Методы расширения
master agentx
agentxsocket /var/agentx/master
extend lld_docker /usr/local/bin/snmp-lld-docker.sh
pass .1.3.6.1.4.1.9999.50 /bin/bash /usr/local/bin/snmp-control.sh
## Настройка Трапов## trapsess: -e [EngineID ПРИЕМНИКА], -n [Имя контекста]
trapsess -v3 -e 0x8000000001020304 -u myuserv3 -l authPriv -a SHA -A AuthPass123 -x AES -X PrivPass123 -n "system_events" 192.168.1.100:162
authtrapenable 1

5. ПОДРОБНЫЙ ИНТЕРФЕЙС КАСТОМНЫХ СКРИПТОВ (EXTEND, PASS, AGENTX)

А. Метод EXTEND (stdout)
Конфиг: extend info_service /usr/local/bin/check_info.sh
Вывод попадает в nsExtendOutputFull.

Б. Метод PASS (LPC)
Вход: $1 (-g/-n/-s), $2 (OID), $3 (Type), $4 (Value).
Пример скрипта /usr/local/bin/snmp-postfix-queue.sh:
#!/bin/bash
OID=".1.3.6.1.4.1.9999.60.1"
if [ "$1" = "-g" ]; then
echo "$OID"; echo "gauge"; find /var/spool/postfix/active -type f | wc -l
fi

В. Метод AgentX (Python)
Файл /usr/local/bin/agentx-app.py:
import pyagentx
class MyUpdater(pyagentx.Updater):
def update(self):
self.set_value('.1.3.6.1.4.1.9999.10.1.0', 'Active', 'string')
if name == "main":
pyagentx.setup(socket='/var/agentx/master')
pyagentx.Agent().register('1.3.6.1.4.1.9999.10', MyUpdater).start()

Г. Сервис Systemd для AgentX:
[Unit]
Description=Python AgentX Sub-agent
After=snmpd.service
[Service]
ExecStart=/usr/bin/python3 /usr/local/bin/agentx-app.py
Restart=always
User=root
[Install]
WantedBy=multi-user.target

6. ИНТЕГРАЦИЯ В ZABBIX: LLD, ТРАПЫ И ПРЕПРОЦЕССИНГ

Использование имен MIB:
На сервере Zabbix выполнить apt install snmp-mibs-downloader && download-mibs. В /etc/snmp/snmp.conf добавить mibs +ALL.
Препроцессинг в Zabbix (JavaScript):

* IP Hex -> Dec: var hex = value.replace(/\s+/g, ''); var ip = []; for (var i = 0; i < hex.length; i += 2) { ip.push(parseInt(hex.substr(i, 2), 16)); } return ip.join('.');
* Матчинг текста: return (value.match(/Error/i)) ? 1 : 0;
* Удаление строк: return value.split('\n').filter(function (line) { return !line.match(/Debug/i); }).join('\n').trim();


7. ГЛУБОКАЯ ДИАГНОСТИКА CLI: КЛЮЧИ И ПАРАМЕТРЫ

А. snmptranslate, snmpget, snmpwalk:

* snmptranslate -Td [OID]: Описание объекта и границ.
* snmpget -d -v3 ...: HEX-дамп пакетов (проверка msgAuthParams и ScopedPDU).
* snmpwalk -Dtsm -v3 ...: Отладка USM (дешифрация).

Б. Анализ трафика (tcpdump):

* tcpdump -i any -nn -vv -X port 161
* Discovery: Пакет с пустым EngineID (0400).
* Report: Пакет от агента с реальным EngineID.
* Header v3: msgFlags 0x03 (authPriv).

В. Управление (snmpset):
snmpset [auth_options] [ip] [oid] [type] [value]
